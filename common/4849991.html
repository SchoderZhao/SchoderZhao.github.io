java公共组件 : 14.钉钉群自定义机器人报警工具

Created by 用户 -cb15a, last modified by 赵兴磊 on 五月 06, 2022

一、群聊机器人添加

1.点击群设置 > 群智能助手

2.点击添加机器人

3.点击自定义 > 添加

4.设置机器人名字和限制的IP，IP填写项目所在服务器的实际IP > 点击创建

5.创建完成后即可获得 webhook，是这个机器人的地址，用于参数的配置


二、项目引入依赖

maven

<dependency>
    <groupId>com.itjcloud.common</groupId>
    <artifactId>common-alert</artifactId>
    <version>正式版https://gitee.com/yuezhix/itjcloud-common/releases (使用正式版)</version>
</dependency>

gradle

implementation "com.itjcloud.common:common-alert:正式版https://gitee.com/yuezhix/itjcloud-common/releases (使用正式版)"

三、项目中在配置文件处配置参数

com:
  itjcloud:
    alert:
      webhook: 步骤一中获得的webhook
      env: 环境名称
      allowAlarm: true为开启报警通知，false为关闭通知

四、项目中在需要的地方传参调用

1.通过@Autowired引入

@Autowired
DingTalkUtil dingTalkUtil;

启动类上需要配置
@ComponentScan(basePackages={"本项目包名","com.itjcloud.common.alert"})

2.在业务代码中使用
//报警名称
String title = "业务异常";
//异常信息
String message = "Errors occurred while build effective model from C:\\Users\\dell\\.gradle\\caches\\modules-2\\files-2.1\\log4j\\log4j\\1.2.15\\1483490805e8f86c21240958629905e733d079eb\\log4j-1.2.15.pom:\n" +
        "    'build.plugins.plugin[io.spring.gradle.dependencymanagement.org.apache.maven.plugins:maven-antrun-plugin].dependencies.dependency.scope' for junit:junit:jar must be one of [compile, runtime, system] but is 'test'. in log4j:log4j:1.2.15";
//@通知人集合

//通知人手机号，如果是空集合就@全体，如果传null就不@
